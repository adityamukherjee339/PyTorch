Tensors are the fundamental data structure of deep learning. They represent multi-dimensional arrays of numbers, optimized for hardware acceleration.

## Tensor Attributes

Every tensor has three key attributes you should always check when debugging:
1. `shape`: The size of the tensor along each dimension.
2. `dtype`: The data type of the tensor's elements (e.g., `torch.float32`).
3. `device`: The hardware device storing the tensor (e.g., `cpu`, `cuda:0`).

```python
import torch

# Creating tensors
x = torch.tensor([[1, 2, 3], [4, 5, 6]], dtype=torch.float32)

print(f"Shape: {x.shape}")
print(f"Datatype: {x.dtype}")
print(f"Device: {x.device}")
```

<Callout type="warn">
Tensor operations will fail if the tensors are on different devices or have mismatched data types. Always ensure device and dtype alignment!
</Callout>

## Reshaping and Squeezing

Manipulating tensor shapes is a daily occurrence in deep learning.

- `reshape()`: Changes the shape of a tensor.
- `view()`: Similar to reshape, but shares memory with the original tensor (must be contiguous).
- `squeeze()`: Removes all dimensions of size 1.
- `unsqueeze()`: Adds a dimension of size 1 at a specified index.
